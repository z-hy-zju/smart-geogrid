This repository contains three Python scripts for processing 36‑channel sensor data, extracting features, and training CNN models to predict the (x, y) coordinates of falling objects, as well as evaluating different 4‑sensor combinations.

─────────────────────────────────────────────────────────────────────────────
File Overview

1. Feature extraction.py
   - Reads raw Excel files (each with 36 columns of time‑series data) from subfolders.
   - For each sensor, computes 14 statistical/spectral features (mean, std, FFT peak, etc.).
   - Saves the features as *()_features.xlsx (14 rows × 36 columns) preserving the original subfolder structure.
   - Input folder: "excel" (configurable) → Output folder: "sensor_features" (configurable).

2. Location Recognition.py
   - Loads the feature files generated by Feature extraction.py.
   - Trains a CNN to predict (x, y) coordinates from the full 504‑feature set (36 sensors × 14 features).
   - Performs data augmentation, feature importance analysis, and a second training on the top‑50 most important features for comparison.
   - Generates training curves, scatter plots with confidence intervals, and importance heatmaps.

3. Different sensor combinations identification.py
   - Loads a predefined list of 4‑sensor combinations from an Excel file (s.xlsx).
   - For each combination, trains a CNN on the selected sensors' features (all 56 features) and separately on a subset of the most important features (top‑50 after importance analysis).
   - Saves per‑combination results (R² scores, metrics) and produces a summary Excel file.
   - Performs data augmentation independently for each combination.

─────────────────────────────────────────────────────────────────────────────
Requirements

Python 3.8+
torch, numpy, pandas, scipy, scikit‑learn, matplotlib, seaborn, openpyxl, xlrd

─────────────────────────────────────────────────────────────────────────────
Data Preparation

1. Raw sensor data (Excel files) must be placed in subfolders. Each Excel file must contain 36 columns (sensors) and any number of rows (time points). The script expects no header row; adjust the code if your files include headers.

2. Run Feature extraction.py first to generate the feature files. The output folder "sensor_features" will contain CSV files (e.g., *_features.csv) – the subsequent scripts rely on these.

3. For Location Recognition.py and Different sensor combinations identification.py, a coordinate file coordinate_set.xlsx is required.

4. For Different sensor combinations identification.py only: an additional Excel file s.xlsx is required. It should contain a column with strings representing 4‑sensor combinations, e.g. "[np.int64(2), np.int64(9), np.int64(18), np.int64(33)]". Each combination will be evaluated separately.

─────────────────────────────────────────────────────────────────────────────
Execution Order

1. Feature extraction
   Edit data_folder and output_folder paths inside the script (if needed), then run:
   python "Feature extraction.py"

2. Location recognition (full sensor set + top‑50 feature selection)
   Ensure the feature CSV files are in the "sensor_features" folder and coordinate_set.xlsx is present, then run:
   python "Location Recognition.py"
   Results are saved under results/timestamp/ (two subfolders: all_features and selected_features).

3. Sensor combination evaluation
   Place the feature CSV files in "sensor_features", ensure coordinate_set.xlsx and s.xlsx are present, then run:
   python "Different sensor combinations identification.py"
   Results are saved under results/timestamp/ with subfolders for each combination (combo_1, combo_2, ...).

─────────────────────────────────────────────────────────────────────────────
Outputs

- Feature extraction: *_features.xlsx files in the output folder.
- Location recognition: training curves, scatter plots, feature importance heatmaps, metrics.json, and a CSV with top‑feature information.
- Sensor combination evaluation: per‑combination training curves, scatter plots, performance metrics, and a final Excel summary (sensor_combinations_results.xlsx).

All plots use Arial fonts; logs are saved as training.log in each result directory.

─────────────────────────────────────────────────────────────────────────────
Notes

- The scripts assume 36 sensors and 14 features per sensor. If your setup differs, adjust the constants accordingly.
- Data augmentation (noise, scaling, shifting) is applied independently for each training run.
- The CNN architectures (CNNModel and SelectedFeaturesCNNModel) are defined inside the scripts and can be modified if needed.
- File paths inside the scripts (e.g., sensor_combinations_file, coordinate_file) may need to be updated before execution.
